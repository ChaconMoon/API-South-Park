# La actualización E {.blog_entry_title}

Lee la nueva entrada de este blog de desarrollo, esta vez hablando sobre la actualización E, la versión 0.2.
{.blog_intro}

## Nueva actualización de la web, presentamos el changelog {.blog_title}

Se ha añadido una nueva sección: ahora puedes ver el changelog de las actualizaciones que vayan saliendo con todos los cambios explicados de forma detallada.

Puedes verlo [aquí](/changelog/Version_0.2).

Este changelog tiene la finalidad de presentar los cambios en la API de forma más profesional y reservar el blog de desarrollo para comentar el desarrollo de forma más cercana.

## Actualización de la API. {.blog_title}

- Se han añadido todos los personajes de la serie cuyo nombre empieza por E.
- Se ha añadido el episodio [329](/api/episodes/329) y los dos personajes que debutaron en él.
- Se han corregido todas las imágenes de familia que faltaban.
- Se han añadido dos familias nuevas: la 71 y la 72.
- Se ha corregido un error en la asignación de la familia de Bebe Stevens.

Para verlo más detalladamente, puedes consultar el [Changelog 0.2](/changelog/Version_0.2).

## Empieza el versionado de la API. {.blog_title}

A partir de ahora se empezará a versionar la API según el contenido que se vaya añadiendo. En este caso, esta es la versión 0.2 y seguirá la siguiente nomenclatura:

- Primer dígito: 0 (Cambiará a 1 una vez la API haya salido de la beta).
- Segundo dígito: 2 (Cambiará con cada actualización importante de contenido de la API, actualización del sitio web, nuevo endpoint o nuevo episodio).
- Tercer dígito: 0 (Cambios menores de datos o correcciones de datos).
- Hotfix: se aplicará un hotfix en caso de que sea necesario corregir algo mediante la base de datos o errores que impidan el acceso a un endpoint o a una sección del sitio.

## Conclusión {.blog_title}

Esta entrada del blog de desarrollo ha sido más corta de lo normal, no porque la actualización sea pequeña, sino porque confío en que el changelog me permita expresar de forma más clara los cambios de cada actualización. Además, el changelog se actualizará de forma retroactiva con los cambios de las versiones anteriores.
